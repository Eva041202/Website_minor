DOM = document object model
Soort kopie van je HTML maar dan geschikt voor JavaScript

AOM = accessibility object model
Bedoeld voor accesive no... zoals screenreaders

Interface = url invullen, return kunnen doen, tussen browsers kunnen schakelen, backbuttons, etc

Browser bestaat uit: HTML, CSS, DOM, AOM, Interface, JavaScript

Parser is een stukje programma dat een string heeft zoals HTML en gaat uitstukken wat er in zit.

Als je de layout veranderd, JavaScript van oh nee toch 3 kolom, maar dit is een zware operatie. Kost veel tijd en resources.
Kijk uit met veranderingen van de layout (boodschap = hou het lekker simple)

Rendering engine is verantwoordelijk voor HTML en CSS parsing en rendering, constructive van de DOM en AOM trees.
Dus NIET interface of JavaScript engine

Rendering engine van browser = centrale deel van browser.
Is alles behalve JavaScript en interface.
Kan op verschillende broswers worden gebruikt.
Meest gebruike rendering engine is Blink (a.k.a Chromium).
Chromium wordt gebruikt door Chrome, Edge, Arc, Opera, Vivaldi en meer (Firefox niet).
Als dus iets in CSS op chrome werkt, werkt het ook voor Edge, Opera en de rest.
Deze browsers gebruiken dus een rendering engine die al bestaat (is makkelijker ofzo).

Browser die Peter Paul Koch benoemde:

Mosaic (1993)
Dit is de eerste browser, zoals wij een browser herkennen.
Zij hebben de tag ontworpen om plaatjes op je browser te krijgen. Zo werden ze bekend bij de gebruikers.

Netscape (1994)
Mosaic bouwde een tweede browser en dat was Netscape.
Voor het gebruik van Netscape moest je betalen. Je kreeg dan wat thuis gestuurd en dat kon je installeren op je computer.
Dit was even een succes tot de komst van Internet Explorer.

Internet Explorer (1995)
Deze browser kon je zomaar downloaden en was gratis.

Opera (1995)
Dit is een wat kleinere browser die op kwam.
Toen der tijd was dit een goede browser.
Één van hun directeuren was ook de uitvinder van CSS.
Met Opera wilde hij laten zien hoe CSS echt werkte.

Internet Explorer 5 for Mac (IE 5 Mac) (2000)
Apple werd hier ondersteund door Microsoft, maar was nog geen goede browser voor de mac.
Deze browser ondersteunde CSS ook erg goed.

Konqueror (2000)
Dit was eerst een hobby-project. Dat bleek een behoorlijk goede browser te zijn.
Het ging steeds beter met Apple en ze hadden ondertussen Microsoft uitgekocht.
Zij hadden een rendering engine genomen die al bestond en deze verbeterd.

Safari (2003)
Safari heeft weer de rendering engine gepakt die Konqueror verbeterd had.

Firefox (2004)
Heeft rendering engine Gecko.
Gecko komt weer van Netscape.

Safari iOS (2007)
Dit was een andere browser dan die van de Mac.
Dit is een mobielere browser met touch.
Nokia en Blackberry hadden al een browser voordat deze Safari uitkwam. De vernieuwde browser van Blackberry pakte de rendering engine van Safari iOS.
Safari iOS is zelfs gebruikt door Google Chrome.

Chrome (2008)
Zoals eerder benoemd, nam Chrome de rendering engine van Safari iOS.
Chrome werd op een gegeven moment gek van Apple, omdat Apple niks nieuws wilde in Webkit.
Zo zijn ze zelf verder gegaan en dat werd Blink. Gecko, de rendering engine van Firefox, gebruikte Chrome niet.
Ze zijn voor die van Konqueror gegaan, want die kwam lichter op gang.

Flow (2020)
Deze browser is gemaakt door een engels bedrijf die browsers maaktten voor tv's.
Ze wilden hun eigen website browser maken.
Deze browser pakt CSS niet altijd goed.

Ladybird (2024)
Dit is gemaakt door een Zweedse man. Hij heeft bij Apple gewerkt.
Hij had een hobby-project nodig om van zijn drugs verslaving af te komen. Dat werd deze browser.

Igalia
Dit is een spaans opensource bedrijf in handen van de werknemers zelf.
Ze maken delen van rendering engines.
Grids die in Chromium zitten zijn gemaakt door Igalia.
Zelf maken ze geen browsers, maar ze weten hier wel heel veel van af.
